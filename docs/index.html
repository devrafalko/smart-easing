Description

Tell me what it is for

The smart-easing library works like CSS animation and transition, except it's controlled with JavaScript.
You can use the smart-easing library:

  • anywhere that CSS animation and transition is limited
  • to control the DOM elements flow via JavaScript
  • to animate the CSS gradients
  • to animate Canvas (eg. the graphs)
  • to display the counting out numbers smoothly
  • to move your autonomous car or steer your rocket with smooth moves

Show me how it works

Check out the easing simulator that shows how the smart-easing works on DOM and Canvas samples with the built-in easing modes.

    Before we start

    The smart-easing library may seem complex, but you really need just 5 minutes to understand how it works and to start using all its features. This section is dedicated to explain all key concepts and ideas, how the smart-easing work, so you could read the documentation without effort and frustration.

    Rerendering number

    The smart-easing library does not move the DOM elements or Canvas animations itself.

    It re-calculates the [Number] value. This value is passed through the render event, that is fired for each frame of animation. This value can be attached to some DOM or Canvas element, to control its flow with the chosen easing mode.

      Animation

      The required smart-easing library returns an object with [Function] Animation and [Function] Blender constructors. The Animation constructor creates a new [Animation] instance. The Animation constructor takes one [Object] [0] argument with some properties to set the initial configuration of the animation. This animation is not run immediately.

        Motion templates

        Before you run the animation, create the motion templates. One motion template is a set of motion properties, that determine the behaviour of this one movement. The whole animation is based on motion templates, that you have defined before the animation start. You combine the motion templates in the chains with the accessible chaining methods and the animation follow this motion templates sequentially when it's running. In the end, the animation time is the sum of all chained motion templates' times. This attitude saves setting the motion properties of every next movement again and again. You can simply use the motion indicator, for example the motion template's identifier, to indicate the motion template that you want to be used in the animation, and you can run one motion template multiple times.

        The motion templates can be defined both in the constructor and with the [Animation].motions.add method.

          Motion Properties

          The motion properties determine how the motion should behave when it is running in the animation. You can set initial and final value|position of the motion, the duration or the speed of this motion, one of 69 accessible easing modes and how many frames should be rendered per one second. You can also define the delay before the motion start and the identifier of this motion template what makes easy then to indicate this template in the accessible chaining methods and control methods.

          The full list of all accessible motion properties is described here.

            Defaults

            When the motion property is not defined for the motion template, the native default value is used for this motion.

            Let's say that you have created a new motion template and did not define the fps motion property. Then the native default value 32 is used for this motion template.

            See the motion properties' native default values in the documentation descriptions.

            Defining each motion property for each motion template can be annoying. If some motion property is common for all (or the majority of) motions in the animation, you don't have to define this motion property for each motion template separately. You can define the user default motion properties. These all user default motion properties will be used for each motion template, that has not got its own ones defined. Thus, the motion property inherits the user default motion property value, rather than the native default motion property value.

            The motion templates' motion properties have the priority over the user default motion properties, and the user default motion properties have the priority over the native default motion properties.

            The user default motion properties can be defined both in the constructor and with the [Animation].defaults setters.

              Chaining Methods

              When the new [Animation] instance is created with some motion templates, it's time to decide how, where, when and how many times each motion template should be used in the animation and, finally, to run the animation. You can use the accessible chaining methods to indicate the desired motion templates to be used in the animation and to set them in the desired execution order.

              The list of all accessible chaining methods: first, last, previous, next, go, chain, all, reverse, boomerang, repeat

              Each chaining method either takes the particular motion template from the motion templates collection itself (eg. first, last, all, boomerang), or lets to indicate the desired motion template, that should be run in the chain, with the motion indicator passed as the argument (eg. go, chain).

              When any chaining method is called, the animation runs, unless the autostart motion property is set to false, then the resume control method must be fired in order to run the animation.

              All chaining methods can be combined in the chains in various ways. When the animation starts, it calls the first chained method with the indicated motion template. After this motion is finished, the next chained method is fired with the indicated motion template(s), etc.

                Motion Indicators

                Some of the chaining methods (and some other methods in the library) require to indicate the motion template(s) as the argument(s). In order to make it easier, you can indicate the chosen motion template in a various number of ways. You can indicate motion template by its identifier, by its index in the motion templates collection, by its [Motion] instance, etc.

                See the full list of Motion Indicators.

                  Events

                  When the animation is pending, it runs the motions in the desired order, what already sounds nice. It would be much nicer though, to have the control over the animation when it is pending.

                  You can use the accessible control events to control the animation when it is in the run and to catch each key moment of the animation. Each control event is accessible as the setter and getter. If you set the [Function] handler to the event, it will be fired in the particular moment of the animation.

                  Inside the function handler you can, for example, control the animation flow with the accessible control methods, or get some data about the pending animation with the [Time] or [Queue] getters.

                  The render event is the most important one, as it lets to make a real use of the smart-easing library. When the render event handler is set, it is called for every frame, thus, it lets to refresh the animation view.

                  The control events can be set in the constructor.
                  The control events can be also set and get via the [Animation] instance.

                    Control methods

                    When the animation is running, it can be controlled with the control methods pause, resume and reset. The pause method pauses the flow in the current position, while resume method continues the animation from the paused position. The reset method moves the flow to the beginning of the animation and starts it again.

                      Modifying motion templates on the run

                      The motion templates are executed with the chaining methods. By default, when the animation is pending, each subsequent motion template is running in accordance with all its motion properties defined.

                      But... it may seem a little bit stiff. In order to make the animation more flexible and convenient, the motion template can be modified in the flow. It may turn out useful if you want to alter the motion template behaviour a little bit just for the single run. The only thing you have to do is to call the chaining method, indicate the desired motion template and define some additional motion properties with the new values, that should be used for this one execution.

                      All possible ways how to modify the motion template in the flow are described in the documentation section.

                      The motion template is modified only for this one execution. The motion properties initially defined for this motion template remain unmodified for its further executions.

                      The motion properties can also be reset. Then, the default motion properties are used for this one execution.

                        Custom motion templates

                        If some motion is repeatable in the animation, it is more convenient to create a motion template. Then it is easy to indicate this motion template multiple times in the chaining methods with the motion indicators.

                        If the motion is meant to be used just once in the animation, there is no a necessity to create a motion template for it. The unique single-use motion can be simply passed through the go or chain chaining method as the [Object] argument with the chosen motion properties defined. Thus, the custom motion is created in the flow, when the chaining method is called. It is then executed just once and not stored in the motion templates collection.

                        The id is unnecessary in the custom motion template.
                        The custom motion creating rules are described in the documentation section.

                          Animation data

                          The [Animation] instance gives the access to some animation data that is generated while the animation is pending.

                          [Animation].motions
                          It gives the access to the [Motions] instance, that allows to navigate through the collection of all created motion templates. With the accessible methods and getters, the motion templates can be found or removed easily, or the new ones can be added.
                          [Animation].state
                          It gives the access to the [State] instance, that allows to figure out, whether the animation is pending, has been started, is already finished, etc.
                          [Animation].queue
                          It gives the access to the [Queue] instance, that stores the data about all motion templates that are used (chained) in this animation.
                          [Animation].time
                          It gives the access to the [Time] instance, that allows to control the time of pending animation and the current motion.

                            Break the animation

                            In order to pause the animation (stop the flow in the current position), you can use the pause control method.

                              In order to reset the animation (stop the flow in the initial position), you can use the combination of reset and pause control methods.

                                In order to break the pending animation and run the new chain of motions, you just call any of the chaining methods (or the chain of chaining methods). It breaks the currently pending animation and starts the new one.

                                  In order to continue the animation when it is finished with the new chain of motions, use the afterAll control event, in which you call the new chain of chaining methods.

                                    Multiple animations

                                    Sometimes, there is a need to recalculate more than one [Number] value at once.

                                    There is a possibility to use [Array] collection of values to be recalculated rather than a singular [Number] value. You have to define the start and stop motion properties in your motion template with the [Array] objects. This [Array] collection of [Number] items will be then passed through the render control event, when the animation is pending, with each [Number] item recalculated to make it smooth, according to the chosen easing mode. These value items can be attached to any DOM or Canvas elements or used in any other way.

                                      Ok, but sometimes there is still a need to recalculate more than one [Number] value at once, that have individual flows, and that must be controlled separately.

                                      Then, you have to create multiple [Animation] instances with the Animation constructor, define individual set of motion templates and control them individually with accessible chaining methods, control events and control methods.

                                        When you run many animations at the same time, the [Blender] instance may turn out useful. It can be used to render all animations at once, with the one rendering event.

                                        It can finally increase the efficiency (eg. of DOM rerendering), as multiple DOM elements are refreshed simultaneously.

                                          See how to use [Blender] instance, to create common render event.

                                          [Animations]

                                          The [Animations] instance is accessible via [Blender].animations getter. It stores the collection of all [Animation] instances that have their render control event referred to this [Blender] instance.

                                          In order to refer the render control event of the animation to the [Blender] instance, use [Blender].id method.

                                          The this keyword inside the [Blender].render [Function] event handler refers to the [Animations] instance.

                                          Thanks to the [Animations] instance, you can get the access to all [Animation] instances that uses the common [Blender] instance to render their easing values.

                                            Methods

                                            • get(String|Animation)

                                              • It takes one [0] argument that indicates the seeking [Animation] instance
                                              • The [String] argument should indicate the identifier of the seeking [Animation] instance, that has been passed through the [Blender].id method, when it was set as the [Animation].render event handler
                                              • The [Animation] argument should indicate this seeking [Animation] instance
                                              • It returns the seeking [Animation] instance, if it has been referred to the [Blender] instance
                                              • It returns null, if the [Animation] instance has not been referred to the [Blender] instance

                                            • has(String|Animation)

                                              • It takes one [0] argument that indicates the seeking [Animation] instance
                                              • The [String] argument should indicate the identifier of the seeking [Animation] instance, that has been passed through the [Blender].id method, when it was set as the [Animation].render event handler
                                              • The [Animation] argument should indicate this seeking [Animation] instance
                                              • It returns [Boolean] true if the seeking [Animation] instance has been referred to the [Blender] instance, otherwise it returns false

                                            Getters

                                            • list [Array]

                                              • It returns the [Array] list of all [Animation] instances that have been referred to the [Blender] instance with the [Blender].id method set as the event handler for their [Animation].render control events

                                            • length [Number]

                                              • It returns the [Number] number of all [Animation] instances that have been referred to the [Blender] instance with the [Blender].id method set as the event handler for their [Animation].render control events

                                                  [Blender]

                                                  See the essential explanation of what the [Blender] instance can be used for here.

                                                  constructor

                                                  The required smart-easing library returns an object with [Function] Animation and [Function] Blender constructors. The Blender constructor creates a new [Blender] instance. The Blender constructor takes one [Function] [0] argument with the render event handler.

                                                  This argument is optional. You can set the render event handler with the [Blender].render setter.

                                                    instance

                                                    The [Blender] instance is created by the Blender constructor. It allows to create one common render control event for multiple animations.

                                                    It can finally increase the efficiency (eg. of DOM rerendering), as multiple DOM elements are refreshed simultaneously, rather than by the individual render control events of each [Animation] instance.

                                                      The chosen [Animation] instances' render control events, rather than have their individual [Function] event handlers set, can refer to the [Blender] instance:
                                                    • In order to pass the [Animation]'s easing value to the [Blender] instance, the [Blender].id method should be used as the [Animation].render event handler
                                                    • [Blender].id should be immediately called with the [String] [0] argument that indicates the identifier of this animation
                                                    • The easing value of this [Animation] instance can be then recognized in the [Blender].render event handler by this identifier
                                                    • When the animation is pending, the easing value is passed through the [Blender].render event handler as the [Object] [0] argument. This object has the property defined, that coheres with the animation's identifier

                                                      Events

                                                      • render [Function|null]

                                                        • It is a common render control event of all [Animation] instances that refer to the [Blender] instance with their render control events
                                                        • It is accessible as the setter|getter
                                                        • It is set as null by default
                                                        • It can be set both in the Blender constructor (passed as the argument) and via [Blender].render setter
                                                        • If the [Function] event handler is set for the [Blender].render event:
                                                          • It is called for each frame of each [Animation] instance that refers to it
                                                          • It provides one [Object] [0] parameter
                                                          • This [Object] parameter stores all easing values of all [Animation] instances that refer to it
                                                          • The properties of this object cohere with the identifiers of [Animation] instances that refere to it
                                                          • The this keyword inside the [Function] event handler refers to the [Animations] instance, that gives the access to all [Animation] instances that refer to it

                                                        Methods

                                                        • id(String)

                                                          • This method allows to combine the [Animation] instances with the [Blender] instance
                                                          • It should be used as the event handler for the [Animation].render control event
                                                          • When the id method is set for the [Animation].render , it should be immediately called with the [String] [0] argument
                                                          • The [String] argument should indicate an identifier of the actual [Animation] instance
                                                          • The easing value of this [Animation] instance will be accessible in the [Blender].render event handler via this identifier
                                                          • If the [Blender].id method is used incorrectly, it is ignored ([Animation] instance is not connected with [Blender] instance)
                                                          • If the [Blender].id method is set for the [Animation].render event and called with the duplicated [String] identifier, this [Animation] instance replaces the [Animation] instance that has been previously connected to the [Blender] instance with the same identifier

                                                          • animations [Animations]

                                                            • See the [Animations] description above

                                                          [Defaults]

                                                          See the essential explanation of what the [Defaults] instance can be used for here.

                                                          • The [Defaults] instance is accessible via [Animation].defaults getter
                                                          • The [Defaults] instance is a collection of user default motion properties
                                                          • Each motion property can be set and get in the [Defaults] instance, except id property (that cannot be common for multiple motion templates)
                                                          • The user default motion properties can be also set in the [Animation] constructor

                                                            • The [Modes] instance is accessible via [Animation].modes getter

                                                            • The [Modes] properties (getters) correspond with the modes' [String] names and the values correspond with the modes' [Array] coordinates

                                                              [Motions]

                                                              • The [Motions] instance is accessible via [Animation].motions getter
                                                              • It stores the collection of motion templates' [Motion] instances defined in the [Animation] constructor and added with [Motions].add method

                                                              Methods

                                                              • add(Object|String|Motion)

                                                                • It adds the new [Motion] to the [Motions] collection
                                                                • The new motion template is added as the last item in the collection
                                                                • If the new [Motion] has the same id as the [Motion] that already exists in the [Motions] collection, this [Motion] is overwritten by the new [Motion] instance
                                                                • It takes one [0] argument:
                                                                  • [Object]
                                                                    • It should be a set of motion properties of the new motion template
                                                                    • The new [Motion] instance with all these motion properties is added to the [Motions] collection
                                                                    • If any motion property is defined incorrectly, the default value is used instead
                                                                  • [String]
                                                                    • It should indicate the id motion property of the new motion template
                                                                    • The new [Motion] instance with all the default motion properties is added to the [Motions] collection
                                                                  • [Motion]
                                                                    • The passed [Motion] instance is simply added to the [Motions] collection
                                                                    • If this [Motion] already exists in the [Motions] collection, it is ignored

                                                                    It can be a [Motion] instance taken from another animation, the custom motion or the [Motion] instance previously removed from the [Motions] collection

                                                                • If no argument or the argument of incorrect type has been passed, it is ignored
                                                                • It returns the added [Motion] instance, or null, if any [Motion] has been added to the [Motions] collection

                                                                • remove(Indicator)

                                                                  • It removes the [Motion] template from the [Motions] collection
                                                                  • It takes one [0] motion indicator argument that indicates the desired motion template
                                                                  • Mind, that if the [Motion] is removed from the [Motions] collection when the animation is pending, the further chaining methods that indicate this motion are omitted
                                                                  • It returns the removed [Motion] instance, or null, if the [Motions] collection does not contain the indicated motion template

                                                                • get(Indicator)

                                                                  • It takes one [0] motion indicator argument that indicates the seeking motion template
                                                                  • It returns the [Motion] instance of the seeking motion template
                                                                  • It returns null if the [Motions] collection does not contain the seeking motion template

                                                                • has(Indicator)

                                                                  • It takes one [0] motion indicator argument that indicates the seeking motion template
                                                                  • It returns [Boolean] true if the [Motions] collection contains the seeking motion template, otherwise it returns false

                                                                Getters

                                                                • list [Array]

                                                                  • It returns the [Array] list of the [String] ids of all [Motion] templates in the [Motions] collection
                                                                  • The motion templates order is retained

                                                                • reversed [Array]

                                                                  • It returns the [Array] list of the [String] ids of all [Motion] templates in the [Motions] collection
                                                                  • The motion templates order is reversed

                                                                • first [Motion|null]

                                                                  • It returns the first [Motion] instance of the [Motions] collection
                                                                  • It returns null if the [Motions] collection is empty

                                                                • last [Motion|null]

                                                                  • It returns the last [Motion] instance of the [Motions] collection
                                                                  • It returns null if the [Motions] collection is empty

                                                                • length [Number]

                                                                  • It returns the [Number] number of all [Motion] templates in the [Motions] collection

                                                                [Motion]

                                                                • The [Motion] instance is created for each motion template:
                                                                  • defined in the constructor
                                                                  • added with the [Animation].motions.add method
                                                                  • passed through the go or chain chaining methods as the custom motion
                                                                • The created [Motion] templates can be accessed via the [Motions] or [Queue] instances

                                                                Methods

                                                                • update(Object|Motion)

                                                                  • It updates the [Motion] with the given new motion properties
                                                                  • Pass the [Object] [0] argument with the new motion properties
                                                                    • the defined motion properties are used to update the [Motion] instance
                                                                    • the incorrect motion properties are ignored
                                                                    • if the defined motion property is set to null, the [Motion]'s motion property is reset (the default motion property will be used for this [Motion] instead)
                                                                    • if the [Motion] is stored in the [Motions] collection and its id motion property is updated, the new id value must be unique compared to the rest of the [Motion] instances in the [Motions] collection. Otherwise it is ignored.
                                                                    • if the [Motion] is stored in the [Motions] collection and its id motion property is reset, it is ignored
                                                                    • Pass the [Motion] [0] argument
                                                                      • the passed [Motion]'s motion properties are used to update this [Motion] instance
                                                                      • the id motion property is not updated

                                                                      Mind, that each motion property of the [Motion] can be updated individually with the [Motion].properties setters.

                                                                    Getters

                                                                    • index [Number]

                                                                      • It returns the [Motion]'s index in the [Motions] collection
                                                                      • It returns null for the [Motion] instance that is not added to the [Motions] collection

                                                                    • id [String]

                                                                      • It returns the id motion property of the [Motion]
                                                                      • If the custom motion is created without the id defined, it returns null

                                                                    • properties [Properties]

                                                                      • It returns the [Properties] object with all final motion properties setters and getters:
                                                                        • the motion properties defined for this motion template
                                                                        • the inherited user default motion properties
                                                                        • the inherited native default motion properties
                                                                      • When any property is set to null, this motion property is reset (the default motion property will be used for this [Motion] instead)
                                                                      • When any property is defined with setter (for the first time) incorrectly, the default value is used instead
                                                                      • When any property is redefined with setter incorrectly, the previously defined value remains in use
                                                                      • If the [Motion] is stored in the [Motions] collection and the new id motion property is set, the new id value must be unique compared to the rest of the [Motion] instances in the [Motions] collection. Otherwise it is ignored.
                                                                      • if the [Motion] is stored in the [Motions] collection and its id motion property is reset, it is ignored

                                                                      • repetition [Number]

                                                                        • It returns the number of actual repetition of the [Motion] in the current animation
                                                                        • It works in accordance with the repeat motion property and the repeat chaining method

                                                                      • delayed [Boolean]

                                                                        • It returns [Boolean] true if the delay motion property is defined for the [Motion] (when it is bigger than 0) , otherwise it returns false

                                                                      [Queue]

                                                                      • The [Queue] instance is accessible via [Animation].queue getter
                                                                      • The chain of chaining methods creates the sequence of the motions in the animation
                                                                      • The [Animation].queue getter gives the data of this motions sequence

                                                                      Getters

                                                                      • index [Number|null]

                                                                        • It returns the [Number] index of the current (pending) motion in the queue
                                                                        • It returns null before the animation is started (also in beforeAll control event)
                                                                        • It returns null after the animation is finished (also in afterAll control event)

                                                                      • current [Motion|null]

                                                                        • It returns the [Motion] instance of the current (pending) motion in the queue
                                                                        • It returns null before the animation is started (also in beforeAll control event)
                                                                        • It returns null after the animation is finished (also in afterAll control event)

                                                                      • next [Motion|Promise|null]

                                                                        • It returns the [Motion] instance of the next motion in the queue (that will be executed right after the current one)
                                                                        • It returns the [Promise] object of the next motion in the queue, when the [Function] motion indicator is passed through the chaining method for the next motion

                                                                          The [Motion] instance of the next motion is unknown till the moment the [Function] motion indicator is called for the next motion. The then method of the returned [Promise] object is called in the moment when the [Motion] instance is already known.

                                                                        • It returns the first motion in the queue before the animation is started
                                                                        • It returns null if the current motion is the last one in the queue or after the animation is finished

                                                                      Chaining methods
                                                                      
			
                                                                      
				See the essential explanation of the chaining 
				methods here.
			
                                                                      
			
                                                                      
				Connect the available chaining methods 
				first, 
				last, 
				previous, 
				next, 
				go, 
				chain, 
				all, 
				reverse, 
				boomerang 
				and repeat 
				to create the chain of motion templates. 
				These motion templates are executed in the sequence.
			
                                                                      
			
                                                                        
		
                                                                        
		
                                                                        
			
                                                                        first()
                                                                        
			
                                                                          
				
                                                                        • 
					It runs the first 
					[0] motion template from the 
					[Motions] 
					collection
				
                                                                          • 
				
                                                                        • 
					If the [Motions] 
					collection is empty, 
					this method is ignored (omitted)
				
                                                                          • 
				
                                                                        • 
					The beforeEach and 
					afterEach 
					control events are not called for the ignored motion templates
				
                                                                          • 
				
                                                                        • 
					The first motion template can be run with some motion properties 
					modified
				
                                                                          • 
			
                                                                        
			
                                                                          
		
                                                                          
		
                                                                          
			
                                                                          last()
                                                                          
			
                                                                            
				
                                                                          • 
					It runs the last motion template 
					from the [Motions] collection
				
                                                                            • 
				
                                                                          • 
					If the [Motions] 
					collection is empty, 
					this method is ignored (omitted)
				
                                                                            • 
				
                                                                          • 
					The beforeEach and 
					afterEach 
					control events are not called for the ignored motion templates
				
                                                                            • 
				
                                                                          • 
					The last motion template can be run with some properties 
					modified
				
                                                                            • 
			
                                                                          
			
                                                                            
		
                                                                            
		
                                                                            
			
                                                                            previous()
                                                                            
			
                                                                              
				
                                                                            • 
					It runs the previous 
					motion template from the 
					[Motions] 
					collection in relation to the currently running motion template from the 
					[Motions] collection
					
                                                                              
						If the animation is pending and the [2] 
						motion template from the [Motions] 
						collection is running, the previous 
						chaining method executes the [1] 
						motion template from the [Motions] 
						collection as the next one.
					
                                                                              
				
                                                                              • 
				
                                                                            • This method is ignored (omitted):
					
                                                                                
						
                                                                              • 
							If the previous method is called as the 
							first one in the chain
						
                                                                                • 
						
                                                                              • 
							If the first 
							[0] motion template from the 
							[Motions] 
							collection is running and the previous 
							method is called as the next one in the chain
						
                                                                                • 
						
                                                                              • 
							If the custom motion 
							is running and the previous method 
							is called as the next one in the chain
						
                                                                                • 
					
                                                                              
					
                                                                                
				
                                                                                • 
				
                                                                              • 
					The beforeEach 
					and afterEach 
					control events are not called for the ignored motion templates
				
                                                                                • 
				
                                                                              • 
					The previous motion template can be run with some properties 
					modified
				
                                                                                • 
			
                                                                              
			
                                                                                
		
                                                                                
		
                                                                                
			
                                                                                next()
                                                                                
			
                                                                                  
				
                                                                                • 
					It runs the next motion template from the 
					[Motions] 
					collection in relation to the currently running motion template from the 
					[Motions] collection
					
                                                                                  
						If the animation is pending and the [2] 
						motion template from the [Motions] 
						collection is running, the next 
						chaining method executes the [3] 
						motion template from the [Motions] 
						collection as the next one.
					
                                                                                  
				
                                                                                  • 
				
                                                                                • This method is ignored (omitted):
					
                                                                                    
						
                                                                                  • 
							If the next method is called as the 
							first one in the chain
						
                                                                                    • 
						
                                                                                  • 
							If the last motion template from the 
							[Motions] 
							collection is running and the next method 
							is called as the next one in the chain
						
                                                                                    • 
						
                                                                                  • 
							If the custom motion 
							is running and the next method is 
							called as the next one in the chain
						
                                                                                    • 
					
                                                                                  
					
                                                                                    
				
                                                                                    • 
				
                                                                                  • 
					The beforeEach and 
					afterEach 
					control events are not called for the ignored motion templates
				
                                                                                    • 
				
                                                                                  • 
					The next motion template can be run with some properties 
					modified
				
                                                                                    • 
			
                                                                                  
			
                                                                                    
		
                                                                                    
		
                                                                                    
			
                                                                                    go()
                                                                                    
			
                                                                                      
				
                                                                                    • 
					It runs the chosen motion template from the 
					[Motions] 
					collection
				
                                                                                      • 
				
                                                                                    • 
					The [0] 
					argument 
					should be passed with the 
					motion indicator 
					that indicates the desired motion template
				
                                                                                      • 
				
                                                                                    • 
					The incorrect motion indicator is ignored 
					(the next motion in the queue is called)
				
                                                                                      • 
				
                                                                                    • 
					The beforeEach 
					and afterEach 
					control events are not called for the ignored motion templates
				
                                                                                      • 
				
                                                                                    • 
					The indicated motion template can be run with some properties 
					modified
				
                                                                                      • 
				
                                                                                    • 
					The go method allows to 
					run the custom motion
				
                                                                                      • 
			
                                                                                    
			
                                                                                      
		
                                                                                      
		
                                                                                      
			
                                                                                      chain()
                                                                                      
			
                                                                                        
				
                                                                                      • 
					It runs the list 
					of chosen motion template from the 
					[Motions] 
					collection in the queue
				
                                                                                        • 
				
                                                                                      • 
					The motion templates should be indicated with the 
					motion indicators:
					
                                                                                          
						
                                                                                        • 
							as the following [0], 
							[1], [2], 
							etc. arguments
						
                                                                                          • 
						
                                                                                        • 
							as the items of [Array] 
							object passed as the [0] 
							argument
						
                                                                                          • 
					
                                                                                        
					
                                                                                          
				
                                                                                          • 
				
                                                                                        • 
					The incorrect motion indicator is ignored 
					(the next motion in the queue is called)
				
                                                                                          • 
				
                                                                                        • 
					The beforeEach 
					and afterEach 
					control events are not called for the ignored motion templates
				
                                                                                          • 
				
                                                                                        • 
					The indicated motion templates can be run with some properties 
					modified
				
                                                                                          • 
				
                                                                                        • 
					The chain method allows to run the 
					custom motions
				
                                                                                          • 
				
                                                                                        •

					method to repeat the 
					chain of motion templates 
					rather than each motion template separately
				
                                                                                          •

                                                                                        • 
					It runs all motion templates from the 
					[Motions] 
					collection from the first one to the 
					last one
				
                                                                                          • 
				
                                                                                        • 
					The first and|or 
					last motion template can be 
					trimmed 
					from the queue
				
                                                                                          • 
				
                                                                                        • 
					All 
					motion templates can be run with some properties 
					modified
				
                                                                                          • 
				
                                                                                        • 
					Use repeat 
					method to repeat the 
					chain of all motion templates 
					rather than each motion template separately
				
                                                                                          • 
			
                                                                                        
			
                                                                                          
		
                                                                                          
		
                                                                                          
			
                                                                                          reverse()
                                                                                          
			
                                                                                            
				
                                                                                          • 
					It runs all motion templates 
					from the [Motions] 
					collection from the last one to 
					the first one
				
                                                                                            • 
				
                                                                                          • 
					The first and|or 
					last motion template can be 
					trimmed from the queue
				
                                                                                            • 
				
                                                                                          • 
					All 
					motion templates can be run with some properties 
					modified
				
                                                                                            • 
				
                                                                                          • 
					Use repeat 
					method to repeat the 
					chain of all motion templates 
					rather than each motion template separately
				
                                                                                            • 
			
                                                                                          
			
                                                                                            
		
                                                                                            
		
                                                                                            
			
                                                                                            boomerang()
                                                                                            
			
                                                                                              
				
                                                                                            • 
					It runs all motion templates 
					from the [Motions] 
					collection from the first one to the 
					last one and back
				
                                                                                              • 
				
                                                                                            • 
					The first and|or 
					last motion template can be 
					trimmed from the queue
				
                                                                                              • 
				
                                                                                            • 
					All 
					motion templates can be run with some properties 
					modified
				
                                                                                              • 
				
                                                                                            • 
					Use repeat 
					method to repeat the 
					chain of all motion templates 
					rather than each motion template separately
				
                                                                                              • 
			
                                                                                            
			
                                                                                              
		
                                                                                              
		
                                                                                              
			
                                                                                              repeat()
                                                                                              
			
                                                                                                
				
                                                                                              • It works in combination with 
					chain, 
					all, 
					reverse and 
					boomerang chaining methods
				
                                                                                                • 
				
                                                                                              • 
					While repeat 
					motion property repeats 
					one motion template itself, 
					the repeat 
					chaining method repeats the 
					chain of motion templates
				
                                                                                                • 
				
                                                                                              • 
					It takes one [0] 
					argument that indicates the 
					number of repetitions
					
                                                                                                  
						
                                                                                                • 
							it must be a [Number] 
							positive integer or Infinity
						
                                                                                                  • 
						
                                                                                                • 
							if any (or incorrect) 
							argument has been passed, 
							the default 1 value is used instead
						
                                                                                                  • 
						
                                                                                                • 
							the value lower than 1 
							will be increased to 1
						
                                                                                                  • 
						
                                                                                                • 
							the decimal fraction will be rounded 
							to the nearest integer
						
                                                                                                  • 
					
                                                                                                
				
                                                                                                • 
				
                                                                                              • It must be chained with the 
					chain, 
					all, 
					reverse or 
					boomerang chaining method

                                                                                                Control methods
                                                                                                
			
                                                                                                
				See the essential explanation of the control 
				methods here.
			
                                                                                                
			
                                                                                                  
				
                                                                                                • 
					Use the accessible 
					pause, 
					resume and 
					reset 
					control methods to control the flow of the pending animation
				
                                                                                                  • 
				
                                                                                                • 
					the control methods do not take any 
					arguments
				
                                                                                                  • 
			
                                                                                                
			
                                                                                                
				In order to start the animation, 
				call one of the chaining methods 
				(or the chain of chaining methods).
			
                                                                                                
		
                                                                                                
		
                                                                                                
			
                                                                                                pause()
                                                                                                
			
                                                                                                  
				
                                                                                                • It pauses the animation in the particular frame when the animation is pending
                                                                                                  • 
				
                                                                                                • It is ignored, if the animation is already paused, finished or not started yet
                                                                                                  • 
			
                                                                                                
			
                                                                                                
				In order to figure out whether the animation is pending, use 
				
					[State].pending
				 getter.
			
                                                                                                
			
                                                                                                  
		
                                                                                                  
		
                                                                                                  
			
                                                                                                  resume()
                                                                                                  
			
                                                                                                    
				
                                                                                                  • It resumes the animation from the frame on which it was paused
                                                                                                    • 
				
                                                                                                  • It is ignored, if the animation is pending and is not paused
                                                                                                    • 
			
                                                                                                  
			
                                                                                                  
				In order to figure out whether the animation is paused, use 
				
					[State].paused
				 getter.
			
                                                                                                  
			
                                                                                                    
		
                                                                                                    
		
                                                                                                    
			
                                                                                                    reset()
                                                                                                    
			
                                                                                                      
				
                                                                                                    • 
					It runs the animation from the beginning 
					(from the first motion template in the chain)
				
                                                                                                      • 
			
                                                                                                    
			
                                                                                                    
				In order to reset the animation without starting it, 
				combine reset control method 
				with the pause 
				control method or set the 
				autostart 
				motion property to false 
				for the first motion template in the chain.
			
                                                                                                    
			
                                                                                                      
		
                                                                                                      
		
                                                                                                      
			
                                                                                                      Control events
                                                                                                      
			
                                                                                                      
				See the essential explanation of the control 
				events here.
			
                                                                                                      
			
                                                                                                        
				
                                                                                                      • 
					each control event is a 
					getter|setter 
					property of the 
					[Animation] 
					instance
				
                                                                                                        • 
				
                                                                                                      • 
					it can be set in the 
					constructor 
					or with the 
					[Animation] 
					instance setter 
					(see below)
				
                                                                                                        • 
				
                                                                                                      • 
					the control event must be set with 
					[Function] event handler or 
					null
				
                                                                                                        • 
				
                                                                                                      • 
					the getter 
					returns 
					null if the 
					control event has not been set 
					with the [Function] event handler
				
                                                                                                        • 
				
                                                                                                      • 
					the getter 
					returns 
					the previously set 
					[Function] event handler 
					if the control event is re-set 
					with the type-incorrect value
				
                                                                                                        • 
			
                                                                                                      
			
                                                                                                        
		
                                                                                                        
		
                                                                                                        
			
                                                                                                        beforeAll
                                                                                                        
			
                                                                                                        
				
					
                                                                                                        
						Type:

                                                                                                                Custom motions
                                                                                                                
			
                                                                                                                
				See the essential explanation of the custom 
				motion templates here.
			
                                                                                                                
			
                                                                                                                  
				
                                                                                                                • 
					The go 
					and chain 
					chaining methods can
					
                                                                                                                    
						
                                                                                                                  • 
							either indicate 
							the motion template from the 
							[Motions] 
							collection, using the 
							motion indicator
						
                                                                                                                    • 
						
                                                                                                                  • 
							or create the new single use 
							custom motion template in the flow 
							(when the chaining method is called)
						
                                                                                                                    • 
					
                                                                                                                  
				
                                                                                                                  • 
				
                                                                                                                • 
					As the custom motion template is meant to be a 
					single use 
					motion template, it is not stored in the 
					[Motions] 
					collection and thereby
					
                                                                                                                    
						
                                                                                                                  • 
							is not accessible via the 
							
								[Animation].motions
							 getter
						
                                                                                                                    • 
						
                                                                                                                  • 
							cannot be indicated in the further 
							chaining methods by the 
							motion indicators
						
                                                                                                                    • 
					
                                                                                                                  
				
                                                                                                                  • 
				
                                                                                                                • 
					The custom motion template can be passed through the 
					go or 
					chain 
					method as the [Object] object with 
					motion properties defined
				
                                                                                                                  • 
				
                                                                                                                • 
					Both custom motion template 
					and the motion indicator 
					(object with motion template id) 
					can be passed as [Object] 
					[0] 
					argument through the 
					go or 
					chain 
					chaining method. The 
					id 
					motion property determines 
					whether the passed object should be interpreted as the:
					
                                                                                                                    
						
                                                                                                                  • custom motion template
							
                                                                                                                      
								
                                                                                                                    • 
									the id 
									motion property should not be defined 
									(the 
										id 
										motion property is unnecessary in the custom motion 
										template as it is not stored in the 
										[Motions] 
										collection)
									
								
                                                                                                                      • 
								
                                                                                                                    • 
									the id 
									motion property should be unique 
									
										(it does not indicate any motion template of the same id in the 
										[Motions] 
										collection)
									
								
                                                                                                                      • 
							
                                                                                                                    
							
                                                                                                                      
						
                                                                                                                      • 
						
                                                                                                                    • motion indicator
							
                                                                                                                        
								
                                                                                                                      • 
									the id 
									motion property should indicate the motion template of the same 
									id 
									in the [Motions] 
									collection
								
                                                                                                                        • 
								
                                                                                                                      • 
									the new [Motion] 
									is not created then
								
                                                                                                                        • 
								
                                                                                                                      • 
									the indicated motion template from the 
									[Motions] 
									collection is used in the chain instead
								
                                                                                                                        • 
							
                                                                                                                      
							
                                                                                                                        
						
                                                                                                                        • 
					
                                                                                                                      
				
                                                                                                                      • 
				
                                                                                                                    • 
					the [Motion] 
					instance of the custom motion template is accessible via the 
					[Queue] 
					getter
				
                                                                                                                      • 
				
                                                                                                                    • 
					The custom motion template also inherits 
					the default motion properties

                                                                                                                      Modifying motion templates
                                                                                                                      
			
                                                                                                                      
					See the essential explanation of motion templates 
					modifying here.
				
                                                                                                                      
			
                                                                                                                        
				
                                                                                                                      • 
					The motion template's motion properties 
					can be modified in the flow for the one call
					
                                                                                                                          
						
                                                                                                                        • 
							pass the [Object] object 
							with the chosen motion properties through the chaining method
						
                                                                                                                          • 
						
                                                                                                                        • 
							each motion property should be defined with 
							the new value
						
                                                                                                                          • 
						
                                                                                                                        • 
							the incorrect motion properties are ignored 
							and the previously defined values remain in use
						
                                                                                                                          • 
						
                                                                                                                        • these motion properties will be used for this one motion run
                                                                                                                          • 
					
                                                                                                                        
				
                                                                                                                        • 
				
                                                                                                                      • 
					The motion template's motion properties 
					can be reset in the flow for the one call
					
                                                                                                                          
						
                                                                                                                        • 
							pass the [Object] object with 
							the chosen motion properties through the chaining method
						
                                                                                                                          • 
						
                                                                                                                        • 
							each motion property should be defined with the 
							null
						
                                                                                                                          • 
						
                                                                                                                        • 
							the default values will 
							be used for these motion properties in this one motion run instead
						
                                                                                                                          • 
					
                                                                                                                        
				
                                                                                                                        • 
				
                                                                                                                      • 
					The motion templates are modified only 
					for the one execution. 
					The [Motion]'s 
					motion properties remain unmodified 
					for the further calls
				
                                                                                                                        • 
				
                                                                                                                      • 
					In order to update the 
					[Motion]'s 
					motion properties, use the 
					
						[Motion].update
					 method
				
                                                                                                                        • 
				
                                                                                                                      • 
					The motion templates modifying differs depending 
					on which chaining method is used 
					(see below)
				
                                                                                                                        • 
			
                                                                                                                      
			
                                                                                                                      
				It may turn out useful if you want to alter 
				the motion template behaviour just for the 
				single run.
			
                                                                                                                      
			
                                                                                                                        
				
                                                                                                                      • 
					
                                                                                                                        
						
                                                                                                                        
							first, 
							last, 
							previous and 
							next method
						
                                                                                                                        
					
                                                                                                                        
					
                                                                                                                        
						
                                                                                                                          
							
                                                                                                                        • 
								The first, 
								last, 
								previous and 
								next 
								chaining method does not take any argument by default

                                                                                                                          • 
							
                                                                                                                        • 
								The [Object] 
								[0] 
								argument 
								passed through these chaining methods 
								will be used to modify this motion template
							
                                                                                                                          • 
							
                                                                                                                        • 
								The id 
								motion property is ignored
							
                                                                                                                          •

									Pass the [Object] 
									[0] 
									argument 
									with the modified motion properties
									
                                                                                                                              
										
                                                                                                                            • 
											the id 
											motion property must be defined in the passed object to 
											indicate the chosen motion template
										
                                                                                                                              • 
										
                                                                                                                            • the other motion properties are used to modify the indicated motion behaviour
                                                                                                                              • 
									
                                                                                                                            
									
                                                                                                                              
								
                                                                                                                              • 
								
                                                                                                                            • 
									Pass the [Object] 
									[1] 
									argument 
									with the modified motion properties
									
                                                                                                                                
										
                                                                                                                              • 
											the [1] argument 
											object should have the motion properties defined to modify the motion behaviour
										
                                                                                                                                • 
										
                                                                                                                              • 
											the [0] 
											argument 
											should be the motion indicator
										
                                                                                                                                • 
										
                                                                                                                              • 
											the id 
											motion property defined in the [Object] 
											[1] argument is 
											ignored

                                                                                                                              chain method

								Pass the chain of [Object] 
								motion indicators 
								
									(object with motion template 
									id
									)
								
							
                                                                                                                                • 
							
                                                                                                                              • 
								The id 
								motion property in [Object] 
								motion indicator 
								indicates the desired motion template
							
                                                                                                                                • 
							
                                                                                                                              • The other motion properties are used to modify the indicated motion behaviour
                                                                                                                                • 
						
                                                                                                                              
						
                                                                                                                                
					
                                                                                                                                
				
                                                                                                                                •

							reverse and 
							boomerang method
						
                                                                                                                                
					
                                                                                                                                
					
                                                                                                                                
						
                                                                                                                                  
							
                                                                                                                                • 
								The all,

								the motion templates to the 
								[Motions] collection
							
                                                                                                                                    • 
							
                                                                                                                                  •

								template can be indicated by its index in the 
								[Motions] collection
							
                                                                                                                                    • 
							
                                                                                                                                  • 
								The [Number] 
								index of the motion template should be passed as the 
								motion indicator

								motion property

                                                                                                                                    • 
								The seeking 
								[Motion]

								id 
								motion property 
								(either defined by the user, or generated automatically)
							
                                                                                                                                        • 
							
                                                                                                                                      •

								[Motions]

												for automatically generated id (when the id is required)

                                                                                                                                          mode
                                                                                                                                          
			
                                                                                                                                          
				
					
                                                                                                                                          
						Type:
						[String|Array]
					
                                                                                                                                          
					
                                                                                                                                          
						Default:
						"linear"
					
                                                                                                                                          
				                                                                                                                                          
			
                                                                                                                                          

			
                                                                                                                                          
				
                                                                                                                                            
					
                                                                                                                                          • The list of all built-in easing modes is accessible:
						
                                                                                                                                              
							
                                                                                                                                            • 
								via the [Modes] instance
							
                                                                                                                                              • 
							
                                                                                                                                            • 
								in the easing simulator
							
                                                                                                                                              • 
							
                                                                                                                                            • 
								in the library 
								on GitHub
							
                                                                                                                                              • 
						
                                                                                                                                            
					
                                                                                                                                            • 
				
                                                                                                                                          
			
                                                                                                                                          

			
                                                                                                                                            
				
                                                                                                                                          • 
					The smart-easing library uses the 
					De Casteljau's algorithm 
					to draw a Bézier curve of the flow
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					The [String] mode 
					should indicate the one of accessible 
					built-in easing modes
					
                                                                                                                                              
						
                                                                                                                                            • 
							The mode motion property 
							is case insensitive
							
                                                                                                                                              
								Both 'linear' 
								and 'LiNeAr' indicates the same mode
							
                                                                                                                                              
						
                                                                                                                                              • 
					
                                                                                                                                            
				
                                                                                                                                            • 
				
                                                                                                                                          • The [Array] mode should contain the values that are interpreted as the custom easing mode
					
                                                                                                                                              
						
                                                                                                                                            • 
							The [Array] object takes the 
							[Number] value pairs - 
							at least 4 values and 
							at most 34 values
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							Each pair represents the x and 
							y coordinate 
							of the point that builds the curve
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							The first {x:0, y:0} 
							pair is automatically attached
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							The x coordinate 
							must be the [Number] 
							value from 0 to 
							1 
								(the last x 
								always must be 1)
							
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							the y coordinate must be 
							the [Number] value from 
							-1 to 2
						
                                                                                                                                              • 
					
                                                                                                                                            
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					To understand how the 
					Bézier curve is built out of the [Array] 
					values or in order to build your own 
					custom easing mode easily, visit the 
					easing simulator page
					
                                                                                                                                              
						
                                                                                                                                            • 
							choose the built-in mode 
							from the list and see how the curve is drawn
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							see how the chosen mode 
							behaves with the DOM samples
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							copy the generated [Array] 
							coordinates and use it in your animation
						
                                                                                                                                              • 
					
                                                                                                                                            
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					The mode motion property defined in the 
					[Defaults] 
					instance applies to all 
					motion templates by default 
					
						(See the description)

                                                                                                                                          fps
                                                                                                                                          
			
                                                                                                                                          
				
					
                                                                                                                                          
						Type:
						[Number]

						32

					Set the number of frames 
					to be rendered per 1 second
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					The higher fps 
					values make the animation smoother

                                                                                                                                          • 
					It must be a [Number] integer 
					from 1 to 75
				
                                                                                                                                            • 
				
                                                                                                                                          •

					75 value respectively.

					[Defaults] instance

					(See the description)

						Type:

						false
					
                                                                                                                                          
				                                                                                                                                          
			
                                                                                                                                          
			
                                                                                                                                            
				
                                                                                                                                          • 
					If the chaining methods 
					are combined and the multiple motion templates 
					are executed in the sequence with different

					set to true attempts to 
					smooth the switch between the motions in the chain

					mode
				
                                                                                                                                            • 
				
                                                                                                                                          •

					[Defaults] instance 
					applies to all 
					motion templates by default

                                                                                                                                          start, stop
                                                                                                                                          
			
                                                                                                                                          
				
					
                                                                                                                                          
						Type:
						[Number|Array|Function]
					
                                                                                                                                          
					
                                                                                                                                          
						Default:
						
							
                                                                                                                                          
								
									
                                                                                                                                          
										start
										
											
                                                                                                                                          
												
													
                                                                                                                                          
														
															0    for the 
															first motion template in the chain
														
													
                                                                                                                                          
													
                                                                                                                                          
														
															the stop value of the 
															previous motion template 
															in the chain for the second 
															(and further) 
															motion template in the chain
														
													
                                                                                                                                          
												                                                                                                                                          
											
                                                                                                                                          
										                                                                                                                                          		
									
                                                                                                                                          
									
                                                                                                                                          
										stop
										
											the start value of 
											this motion template
										
									
                                                                                                                                          
								                                                                                                                                          
							
                                                                                                                                          
						                                                                                                                                          		
					
                                                                                                                                          
				                                                                                                                                          
			
                                                                                                                                          
			
                                                                                                                                            
				
                                                                                                                                          • 
					The motion flow starts from the start 
					value and ends with the stop value
				
                                                                                                                                            • 
				
                                                                                                                                          • The recomputed value is accessible
					
                                                                                                                                              
						
                                                                                                                                            • 
							in the render control event
						
                                                                                                                                              • 
						
                                                                                                                                            • 
							via the 
								[State].coords
							 getter
						
                                                                                                                                              • 
					
                                                                                                                                            
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					The start and stop 
					motion properties can be defined with the following types:
					
                                                                                                                                              
						
                                                                                                                                            • [Number]
							
                                                                                                                                                
								
                                                                                                                                              • The positive and negative integer and decimal fraction is allowed
                                                                                                                                                • 
								
                                                                                                                                              • 
									The -Infinity, 
									Infinity and 
									NaN is forbidden
								
                                                                                                                                                • 
								
                                                                                                                                              • 
									This recalculated [Number] 
									value is passed through the 
									render 
									control event
								
                                                                                                                                                • 
							
                                                                                                                                              
						
                                                                                                                                              • 
						
                                                                                                                                            • [Array]
							
                                                                                                                                                
								
                                                                                                                                              • It allows to recalculate multiple easing values
                                                                                                                                                • 
								
                                                                                                                                              • 
									The [Array] object 
									must contain [Number] items
								
                                                                                                                                                • 
								
                                                                                                                                              • 
									Each item must respect the [Number] 
									type rules (described above)
								
                                                                                                                                                • 
								
                                                                                                                                              • 
									The [Array] object must contain 
									at least 1 and at most 
									16 [Numer] 
									items (the excess items will be ignored)
								
                                                                                                                                                • 
								
                                                                                                                                              • The incorrect items are ignored and replaced with the default values
                                                                                                                                                • 
								
                                                                                                                                              • 
									This [Array] object with recalculated 
									[Number] items is passed through the 
									render 
									control event
								
                                                                                                                                                • 

							
                                                                                                                                              
						
                                                                                                                                              • 
						
                                                                                                                                            • [Function]
							
                                                                                                                                                
								
                                                                                                                                              • 
									This function must return either 
									[Number] value or 
									[Array] object 
									(described above)
								
                                                                                                                                                • 
								
                                                                                                                                              • 
									The incorrect returned 
									value|object is ignored and replaced with the default value
								
                                                                                                                                                • 
								
                                                                                                                                              • 
									It allows to determine the start or 
									stop motion property's value 
									right before the actual motion template is executed in the chain
								
                                                                                                                                                • 
								
                                                                                                                                              • 
									The this keyword 
									inside the function refers to the 
									[Animations] 
									instance
								
                                                                                                                                                • 
							
                                                                                                                                              
						
                                                                                                                                              • 
					
                                                                                                                                            
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					If the start and 
					stop motion properties are defined 
					with the different types, 
					the [Number] value is converted into 
					[Array] object with this one 
					[Number] [0] item
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					If the [Array] 
					start and 
					stop motion properties are 
					of the different length, 
					the [Array] object with the less 
					number of items is filled with the items of the default values, 
					so that both start and 
					stop are of equal lengths
				
                                                                                                                                            • 
				
                                                                                                                                          • 
					The start and 
					stop motion properties defined in the 
					[Defaults] 
					instance applies to all 
					motion templates by default 
					
						(See the description)

                                                                                                                                              autostart
                                                                                                                                              
			
                                                                                                                                              
				
					
                                                                                                                                              
						Type:
						[Boolean]
					
                                                                                                                                              
					
                                                                                                                                              
						Default:
						true
					
                                                                                                                                              
				                                                                                                                                              
			
                                                                                                                                              
			
                                                                                                                                                
				
                                                                                                                                              • 
					When the autostart is set to 
					true for the motion template, 
					this motion template runs 
					right after it is called with any 
					chaining method
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					When the autostart is set to 
					false for the motion template, 
					the animation is automatically paused 
					before this motion start. The 
					resume 
					control method must be called manually to run this motion
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The autostart motion property defined in the 
					[Defaults] 
					instance applies to all 
					motion templates by default 
					
						(See the description)
					
					
                                                                                                                                                  
						
                                                                                                                                                • 
							when true, 
							all chained motion templates run automatically
						
                                                                                                                                                  • 
						
                                                                                                                                                • 
							when false, the animation is 
							paused before 
							each chained motion start

                                                                                                                                              time
                                                                                                                                              
			
                                                                                                                                              
				
					
                                                                                                                                              
						Type:
						[Number]
					
                                                                                                                                              
					
                                                                                                                                              
						Default:
						1000
					
                                                                                                                                              
				                                                                                                                                              
			
                                                                                                                                              

			
                                                                                                                                                
				
                                                                                                                                              • 
					Set the duration time 
					of the motion
				
                                                                                                                                                • 
				
                                                                                                                                              • It must be a [Number] positive integer
                                                                                                                                                • 
				
                                                                                                                                              • 
					The 1 value equals 
					1 millisecond, while 
					1000 equals 
					1 second, etc.
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The value lower than 1 
					will be increased to 1. 
					The decimal fraction will be rounded to the nearest integer
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The Infinity is forbidden 
					(the default value will be used instead)
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The time motion property defined in the 
					[Defaults] 
					instance applies to all 
					motion templates by default 
					
						(See the description)
					
			
                                                                                                                                              
			
                                                                                                                                              
				If the motions are chained, the animation 
				time is a sum of all chained motions' durations.
			
                                                                                                                                              
		
                                                                                                                                              
		
                                                                                                                                              
			
                                                                                                                                              speed
                                                                                                                                              
			
                                                                                                                                              
				
					
                                                                                                                                              
						Type:
						[Number]
					
                                                                                                                                              
					
                                                                                                                                              
						Default:
						undefined
					
                                                                                                                                              
				                                                                                                                                              
			
                                                                                                                                              
			
                                                                                                                                                
				
                                                                                                                                              • 
					The time 
					and speed motion properties are used 
					alternately
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The speed motion property 
					has a priority over the 
					time 
					motion property. If the speed is properly defined, 
					the time is ignored
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The [Number] 
					speed motion property indicates 
					what distance 
					
						(between start and 
						stop values)
					 should be covered per 1 second
					
                                                                                                                                                
						for { start: 0, stop: 
						100, speed: 
						10 } the coords increases by 
						10 every 
						1000 milliseconds
						
                                                                                                                                                
						for { start: 0, 
						stop: 100, speed: 
						25 } the coords increases by 
						25 every 
						1000 milliseconds
						
                                                                                                                                                
						for { start: 1.1, 
						stop: 0.5, speed: 
						.05 } the coords decreases by 
						0.05 every 
						1000 milliseconds
					
                                                                                                                                                
				
                                                                                                                                                • 
				
                                                                                                                                              • It must be a [Number] positive value
                                                                                                                                                • 
				
                                                                                                                                              • 
					When the speed motion property 
					is set to Infinity, 
					the whole distance is covered in 1 millisecond
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					In order to figure out the actual 
					duration 
					of the motion, when the speed 
					motion property is defined, use the 
					
						[Time].current.duration
					 getter 
					
						(it recalculates the duration from the 
						speed 
						and the distance values)
					
				
                                                                                                                                                • 
				
                                                                                                                                              • 
					The speed motion property defined in the 
					[Defaults] 
					instance applies to all 
					motion templates by default 
					
						(See the description)
					
			
                                                                                                                                              
			
                                                                                                                                              
				When the speed is defined, 
				the final motion duration depends on the difference between the 
				start and 
				stop values 
				(the distance), while the 
				time 
				motion property makes the motion duration constant 
				(regardless the distance).
			
                                                                                                                                              
			
                                                                                                                                              
				It may turn out more useful than the 
				time 
				property, if the two motion templates of the 
				different distances 
				are chained and the equal 
				speed of the flow should be kept.
